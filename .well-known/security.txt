# Security Reporting

Please report security vulnerabilities by emailing security@ipv6-only.example.com
or using GitHub's private vulnerability reporting feature.

Do NOT report security vulnerabilities through public GitHub issues.

Include:
- Type of vulnerability
- Location in source code
- Steps to reproduce
- Proof of concept (if available)
- Impact assessment

# Response Timeline

- Initial response: Within 48 hours
- Status update: Within 7 days
- Resolution target: Within 90 days (severity-dependent)

# Disclosure Policy

We follow coordinated disclosure:
1. Issue reported privately
2. Fix developed in private
3. Security advisory drafted
4. Fix released with advisory
5. Public disclosure after fix available

# Scope

In scope:
- All code in this repository
- Dependencies we control
- Build and deployment systems
- Documentation affecting security

Out of scope:
- Social engineering
- Physical attacks
- Denial of service (unless leading to code execution)
- Issues in third-party dependencies (report to maintainers)

# Safe Harbor

We support security research conducted in good faith:
- Research on your own systems or with permission
- Do not access data you don't own
- Do not violate privacy or destroy data
- Give us reasonable time to fix issues
- Follow responsible disclosure

Researchers acting in good faith will not face legal action.
